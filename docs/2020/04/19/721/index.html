Air720SL 配置GPLDO6输出电压 及MMC的PGIO控制问题 填坑

使用EVB_Air720SL_A11开发板，固件Luat_V0010_ASR1802S.blf，根据硬件手册

配置GPLDO6的输出电压，使用下面代码

--[[
pmd.ldoset(x,pmd.VLDO6)
x=0时：关闭LDO
x=1时：LDO输出1.8V
x=2时：LDO输出1.9V
x=3时：LDO输出2.5V
x=4时：LDO输出2.8V
x=5时：LDO输出2.9V
x=6时：LDO输出3.1V
x=7时：LDO输出3.3V
x=8时：LDO输出1.7V
]]

所有的x值都试了，测量硬件输出始终为1.8V

后刷新Luat固件为Luat_V0010_ASR1802S_FLOAT.blf

重新进行上述测试，输出正常了。问题解决。


而且在V0008版本，每次下载脚本需要按开发板的重启键，但烧写V0010后下载脚本就不需要了，直接下载。


另外一个问题，根据手册

上述几个引脚是可以作为通用GPIO，电压域是GPLDO6,但实际测试没有作用。原因不清楚，所以不能

作为GPIO引脚用作控制信号输出。
